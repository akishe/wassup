DA Mini Project¶

  • 주제 : 온라인 학습 플랫폼 사용자 행동 패턴 분석
  • 개요 : ``` 이번 프로젝트는 학습 서비스를 이용하는 독자들 중 누가 서비스 구독을 연장할지 예측하기 위한 사전 파악에 초점을 맞추고 있습니다. 우리의 목표는 유저들의 학습 습관, 선호도, 이용 행태 등 다양한 측면을 면밀히 분석하는 것입니다.

이를 위해, 우리는 유저들의 학습 패턴, 서비스 이용 시간, 콘텐츠에 대한 반응, 그리고 이들이 얼마나 자주 서비스를 이용하는지 등 다양한 데이터를 수집하고 분석할 계획입니다. 이 데이터는 유저들이 서비스에 얼마나 만족하고 있는지, 어떤 요소들이 그들의 구독 결정에 영향을 미치는지 이해하는 데 중요한 역할을 할 것입니다. ```

  • 데이터셋 : mini.csv
    • 메타데이터
      • user_id: 사용자의 고유 식별자
      • subscription_duration: 사용자가 서비스에 가입한 기간 (월)
      • recent_login_time: 사용자가 마지막으로 로그인한 시간 (일)
      • average_login_time: 사용자의 일반적인 로그인 시간
      • average_time_per_learning_session: 각 학습 세션에 소요된 평균 시간 (분)
      • monthly_active_learning_days: 월간 활동적인 학습 일수
      • total_completed_courses: 완료한 총 코스 수
      • recent_learning_achievement: 최근 학습 성취도
      • abandoned_learning_sessions: 중단된 학습 세션 수
      • community_engagement_level: 커뮤니티 참여도
      • preferred_difficulty_level: 선호하는 난이도
      • subscription_type: 구독 유형
      • customer_inquiry_history: 고객 문의 이력
      • payment_pattern : 사용자의 지난 3개월 간의 결제 패턴을 10진수로 표현한 값.
        • 7: 3개월 모두 결제함
        • 6: 첫 2개월은 결제했으나 마지막 달에는 결제하지 않음
        • 5: 첫 달과 마지막 달에 결제함
        • 4: 첫 달에만 결제함
        • 3: 마지막 2개월에 결제함
        • 2: 가운데 달에만 결제함
        • 1: 마지막 달에만 결제함
        • 0: 3개월 동안 결제하지 않음
        • target: 사용자가 다음 달에도 구독을 계속할지 (1) 또는 취소할지 (0)를 나타냄
  • 의문점
    • 학습 서비스를 이용하는 독자?
      → 학습 플랫폼이면 독자보다는 이용자나 수강생이라는 워딩을 쓰지 않나?

데이터에서 이상한 점¶

  • 위의 데이터는 payment_type의 결제가 구독을 결제한다는 가설을 지니고 만든 데이터이다.
  • 하지만, payment가 구독이라면 '111'은 지난 3개월간 구독을 결제한 사람이나, 그렇다면 subscription_duration이 1일 수가 없음 ('111'인 사람의 subscription_duration은 최소 3이상이어야 한다)
    => payment와 subscrption은 별개의 무언가
    => 구독 유형을 살펴보자

결론 :¶

  1. 구독과 결제를 별개인 학습 플랫폼이다.
  2. 결제를 하지 않는다 해서 서비스를 이용하는 데 제약이 있는 플랫폼은 아니다. 이는 구독 유형에도 해당된다.
    난이도 선택, 커뮤니티 이용, 이용 시간 제약 등의 제약은 존재하지 않음

가정¶

  1. 결제를 한다면 어떠한 디메리트가 없애주거나, 메리트가 있을 가능성이 있음 (예시로 유튜브처럼 구독은 무료로 하며 댓글 달기, 좋아요, 커뮤니티 이용은 자유로우나, 결제를 하면 이용가능한 프리미엄이 있을 수도)
  2. 결제라는 것이 구독이 아닌 어떠한 콘텐츠를 결제하는 것일 가능성이 있음 (강사와의 대화 같은)
  3. target이 구독을 계속할 사용자... 이건 구독을 연장한다고 볼 수 있는데, 구독을 하지 않는 사람에게 구독을 연장한다는 표현을 사용하지는 않을 것
    -> 학습 플랫폼 사이트 안의, 특정한 서비스에서 가져온 데이터일 듯

grouped = df.groupby('community_engagement_level')['average_login_time']
for name, group in grouped:
    q1 = group.quantile(0.25)
    q3 = group.quantile(0.75)
    iqr = q3 - q1
    lower_bound = q1 - 1.5 * iqr
    upper_bound = q3 + 1.5 * iqr
    outliers = group[(group < lower_bound) | (group > upper_bound)]
    upper_outliers = group[(group > upper_bound)]
    lower_outliers = group[(group < lower_bound)]
    print(f"커뮤티니 참여도 {name}의 이상치 개수 : {len(outliers)}")  # 이상치 : {list(outliers)}")
    print(f"커뮤티니 참여도 {name}의 upper_bound 이상의 이상치 개수 : {len(upper_outliers)}")
    print(f"커뮤티니 참여도 {name}의 lower_bound 아래의 이상치 개수 : {len(lower_outliers)}")
    print('')
